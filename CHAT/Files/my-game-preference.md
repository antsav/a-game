# my-game-preference — Anton's taste north-star

**Purpose:** encode **my** game taste so whatever gets built is something I'll love enough to
polish to the extra mile. `games-researcher` finds a *marketable* concept, `game-monetizer`
makes it *pay* — this makes sure it's a game **I actually want to make.** The best concept
sits at the intersection (and happily my taste — no-timer arrow/sort puzzles + idle pull —
overlaps with what's trending).

**Use this lens for:** any "should the game do X / how should this feel / does this fit me"
decision — core loop, mechanics, difficulty, pacing, feel/juice, progression, modes, art/
audio — or checking a monetization/design choice doesn't betray the calm.

## The taste in one line

**Engagement through absorption, not arousal.** Calm focus + satisfying loops + an intrinsic
urge to reach for more — never stress, urgency, threat, or pressure. Meditative *and*
engaging at once. Single-player puzzle.

## The one test (apply to any mechanic)

> **Does this make me play to GAIN something, with freedom to stop — or to AVOID LOSING
> something, under pressure?**
> Gain + freedom = **keep** (pull). Loss + pressure = **cut** (push).

Coziness check: *if you explained the mechanic out loud, would it feel like generosity/calm,
or like pressure/a trick?* Pressure/trick = cut or redesign.

## Hard gates (non-negotiable — a "no" here means redesign)

- **No timer / countdown / time-pressure** in core play.
- **No punishing fail state** — failure is soft, retry freely, progress persists.
- **No forced urgency / FOMO / pressure** (including in monetization — find a calm alternative).
- **No relentless difficulty ramp** — difficulty oscillates; a breather follows every hard level.
- **The player controls the pace** — pause/stop anytime, can advance time themselves.
- **Pull, not push** — motivate via anticipation/progress/curiosity, never deadline/loss.

## What "go the extra mile" looks like

- **Calm + engaging:** core verb passes the autotelic test (satisfying to repeat 1,000× with
  no reward); ride the **lower edge of the flow channel** (clear goal + instant feedback, gentle
  low ceiling); cozy pillars = safety, abundance, softness; an "order from disorder" / tending
  verb at the core (untangle, sort, clear, merge, build). Replace every removed pressure source
  with a calm pull — don't just delete the timer.
- **The pull (intrinsic "reach more"):** an always-visible next goal; a number/board that **goes
  up, never down**; multi-horizon goals (a 5-second, 5-day, 5-week carrot); consider a
  persistent meta-layer that survives resets (banking, not losing) + offline progress (returning
  is a gift). Optimization, not survival, is the engagement substrate.
- **Difficulty & pacing:** very simple, near-trivial start (confidence streak); **oscillate**
  via Introduce → Test → Breather → Stabilize, envelope drifting up gently; aha + cascade bursts
  in the better levels; soft fail, unlimited retries, graduated hints (~after 3 fails); "if hard,
  make it short." Instrument attempts-per-level + time-to-abandon; playtest externally.
- **Feel (calm-tuned juice):** response < 100 ms; restrained, coherent juice — drain/dissolve/
  bloom, **not** detonation; desaturated palette, soft/musical audio, soft iOS haptics; reserve
  the biggest feedback for board-clears against a quiet baseline; reduced-motion / haptics-off
  toggles.

## When monetizer and this conflict

**The feel wins for the core loop.** Monetize via calm-compatible tools only (rewarded
*acceleration* opt-in, remove-ads, cosmetic/relaxing IAP, gentle non-gating boosters, idle/
piggy-bank "gain" framings). Run monetization through the one test too.

## Current best-fit recommendation

**Arrow/untangle puzzle** — my top love + trending + lowest craft barrier + "stakes without a
clock" built in — optionally fused with a gentle idle/prestige progression and a visibly-
growing board for the "reach more" pull. A small color-sort game is a fine feel-building
warm-up.

> This is *my* taste, not a guess. If something seems like a refined or changed preference vs.
> a passing reaction, **ask me** before treating it as a new rule.
