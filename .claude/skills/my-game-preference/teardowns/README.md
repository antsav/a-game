# teardowns/ — why Anton likes these games

A growing library of design analyses of the specific games Anton loves. Each isolates the
**DNA to extract** for a calm-yet-engaging, no-timer game. When Anton mentions a new game he
enjoys, add a teardown here (and fold durable patterns into `../references/`).

`../preference-profile.md` = the taste (the *what*). These teardowns = the *evidence* and the
extractable mechanics. `../references/genre-dna.md` = the synthesis across them.

## Teardown template

```markdown
# <Game> (<genre>)

**Core loop:** one sentence.
**Source of satisfaction / pull:** what hooks, calmly.
**Engaging without a timer:** where the (non-clock) tension comes from.
**Difficulty / pacing:** how it scales.
**DNA to extract:** the reusable design moves.
**Solo-dev buildability:** HIGH / MEDIUM / LOW + the hard 20%.
**Sources:** URLs + dates.
```
