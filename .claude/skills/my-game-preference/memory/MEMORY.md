# memory/ — dated taste discoveries & design lessons

The skill's lab notebook: dated notes on Anton's evolving taste and design lessons learned —
"this fits / this doesn't," new games he likes, playtest insights. `../preference-profile.md`
is the source of truth; this is where new signals land before (or instead of) being folded in.

When Anton says he likes/dislikes something new, capture it here with the date and *why*, then
update the profile if it's durable. **When unsure if it's a lasting preference, ask Anton.**

## Entry template
```markdown
---
date: YYYY-MM-DD
type: taste-signal | playtest | design-lesson
---
**Signal:** what was observed/said.
**Why it matters:** how it refines the taste profile.
**How to apply:** concrete design implication.
**Related:** [[../preference-profile]] · [[../teardowns/...]] · [[../references/...]]
```
