# Design North-Star — The Decision Filter

The actionable distillation of Anton's taste. Run **every** design/feature/mechanic decision
through this. `preference-profile.md` = what he loves; `references/` = how to achieve it; this
= the quick filter to apply in the moment.

## The one test (apply to any mechanic)

> **Does this make the player play to GAIN something, with freedom to stop — or to AVOID
> LOSING something, under pressure?**
> Gain + freedom = **keep** (pull). Loss + pressure = **cut** (push).

And the coziness check: *if you explained this mechanic out loud, would it feel like
generosity/calm — or like pressure/a trick?* Pressure/trick = cut or redesign.

## Hard gates (a "no" here means redesign — these are non-negotiable)

- [ ] **No timer / countdown / time-pressure** in core play.
- [ ] **No punishing fail state** — failure is soft, retry freely, progress persists.
- [ ] **No forced urgency / FOMO / pressure mechanics** (incl. in monetization — find a
      calm-compatible alternative).
- [ ] **No relentless difficulty ramp** — difficulty oscillates; a breather follows every hard level.
- [ ] **The player controls the pace** — pause/stop anytime, can advance time themselves.
- [ ] **Pull, not push** — motivates via anticipation/progress/curiosity, never deadline/loss.

## Design checklist (serve these to go the extra mile)

**Calm + engaging (the absorption sweet spot)**
- [ ] Core verb passes the **autotelic test** (satisfying to repeat 1,000× with no reward).
- [ ] Rides the **lower edge of the flow channel** — clear goal + instant feedback, gentle low ceiling.
- [ ] Installs the **cozy pillars** — safety, abundance, softness.
- [ ] Every removed pressure source is **replaced** with a calm pull (don't just delete the timer).
- [ ] "Order from disorder" / tending verb at the core (untangle, sort, clear, merge, build).

**The pull (intrinsic "reach more")**
- [ ] An always-visible **next goal**; a number/board that **goes up, never down**.
- [ ] **Multi-horizon goals** (a 5-second, 5-day, 5-week carrot).
- [ ] Consider a **persistent meta-layer that survives resets** (banking, not losing) and
      **offline progress** (returning is a gift) — the idle "pull to reach more."
- [ ] Optimization (not survival) is the engagement substrate.

**Difficulty & pacing**
- [ ] **Very simple start** — first levels teach by playing, near-trivially easy (confidence streak).
- [ ] **Oscillate** difficulty via the 4-phase loop (Introduce → Test → Breather → Stabilize);
      envelope drifts up gently.
- [ ] **Aha + cascade bursts** inside the better levels (the "bursts of satisfaction/challenge").
- [ ] Soft fail, unlimited retries, **graduated hints** (first hint after ~3 fails); "if hard, make it short."
- [ ] Plan to **instrument attempts-per-level + time-to-abandon** and **playtest externally**
      (beat the designer blind spot).

**Feel (calm-tuned juice)**
- [ ] Response **< 100 ms**; satisfying snap/clear feedback.
- [ ] Juice is **coherent and restrained**, not maximal — drain/dissolve/bloom, not detonation.
- [ ] Calm sensory layer — desaturated palette, soft/musical audio, **soft iOS haptics**
      (`.soft`/`.light`/Core Haptics low sharpness), minimal cognitive load.
- [ ] Reserve the biggest feedback for board-clears; quiet baseline so payoffs register.
- [ ] Reduced-motion / haptics-off toggles.

## When skills conflict
If `game-monetizer` (FOMO offers, energy gates, timed events) collides with this north-star,
**the feel wins for the core loop.** Monetize via calm-compatible tools — rewarded-ad
*acceleration* (opt-in), remove-ads, cosmetic/relaxing-content IAP, gentle non-gating boosters,
the idle/piggy-bank "gain" framings — never pressure that betrays the calm. Run monetization
mechanics through the one test above too.
